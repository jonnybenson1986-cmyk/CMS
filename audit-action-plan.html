June 2026
📋 To-Do List — What Still Needs to Be Done
  • Fix Firebase cloud sync connection HIGH Go to Firebase Console → cms-wgac-317ce → Authentication → Sign-in method → Enable Anonymous and Email/Password. Then paste config in Settings → Cloud Sync and click Connect.
  • Create staff accounts in Firebase HIGH In Firebase Console → Authentication → Users → Add user for each staff member (name + email + password). Then they can log in to the CMS with their email.
  • Create Firestore database HIGH In Firebase Console → Firestore Database → Create database → Start in production mode → Choose a region close to Australia (australia-southeast1).
  • Test sync across two devices HIGH Once Firebase is connected, add a client on one device and check it appears on another. This confirms real-time sync is working.
  • Enter existing client records into the CMS MEDIUM Transfer active client information from paper files or spreadsheets into the CMS client profiles. Prioritise active/high-risk clients first.
  • Train all staff on how to use the app MEDIUM Run a 30-minute walkthrough: logging in, adding a client, writing a case note, logging time. Each staff member should install the app on their phone.
  • Install the app on all staff phones MEDIUM iPhone: Open the site in Safari → Share button → Add to Home Screen. Android: Open in Chrome → three-dot menu → Add to Home Screen.
  • Set up Firestore security rules MEDIUM In Firebase Console → Firestore → Rules — restrict read/write so only authenticated staff (not the public) can access client data. This is a legal requirement under the Privacy Act.
  • Confirm data backup process MEDIUM Firebase automatically backs up data, but a process should be documented. Decide how often data is exported (e.g. monthly) and who is responsible.
  • Review what client information is collected MEDIUM Under APP 3 (collection), only collect what is needed. Review each field in the client form and remove or make optional anything not required for service delivery.
  • Document who has access to the system LOW Maintain a register of staff who have CMS access. Remove access promptly when staff leave. Review quarterly.
